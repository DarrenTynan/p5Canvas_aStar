A* Search Algorithm

In A* we begin with a Source and a Target node, openSet and closeSet. We also use a temporary array for the neighboring nodes.

From the 'source' node we check all neighbors: North, South, East and West.

Each of these neighbors is marked as visited, and removed from the frontier queue.

We continue iterating until we have searched all nodes or we break early if 'target found'.

The source node is blue, with the target node being red, and the wall being black.

After starting the search with 'Go For It!' the green nodes represent all the nodes on the frontier that have been checked. And if a path is found, this will be show in teal.

You can change the grid size and wall frequency via a dropdown or just Update Grid for a new random grid.

Please note: The animation has been slowed, so, you have a chance to see the process.

g = the movement cost to move from the starting point to a given square on the grid, following the path generated to get there.

h = the estimated movement cost to move from that given square on the grid to the final destination. This is often referred to as the heuristic, which is nothing but a kind of smart guess. We really don’t know the actual distance until we find the path, because all sorts of things can be in the way (walls, water, etc.). There can be many ways to calculate this ‘h’ which are discussed in the later sections.

init openSet (contains calculated f cost nodes)
init closesSet (contains valueated nodes)
add start node to openSet

main loop

  • currentNode = node in the openSet with lowest f
  • remove currentNode from openSet
  • add currentNode to closeSet

  • if currentNode is the target node
    return

  • for each neighbor of the currentNode

  • if neighbor is not traversable or neighbor is in closeSet

    • skip to the next neighbor

  • if new path to neighbor is shorter OR neighbor is not in openSet
    set f of neighbor
    set parent of neighbor to currentNode
  • if neighbor is not in openSet
    add neighbor to openSet
